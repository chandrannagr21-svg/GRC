Arduino Buzzer Project

My Arduino Buzzer Project

Project Overview

In this project, I created and ran a simple program on an Arduino to control a buzzer. My goal was to understand how to use digital pins to trigger an active or passive buzzer and verify the result using the Arduino IDE Serial Monitor. This experience taught me about Arduino coding, wiring, and observing hardware responses.

My Experience

While creating this project, I learned to:

  • Write Arduino code to control a buzzer using digitalWrite() or tone().
  • Wire the buzzer correctly to the Arduino pins and ground.
  • Use the Serial Monitor to print and verify buzzer activity.
  • Troubleshoot common errors like no sound or wrong pin configuration.

Overall, it was a hands-on experience that helped me connect programming logic with physical output.

Arduino Buzzer Code

Here is the code I used to execute the buzzer:

#include <Arduino.h>

const int buzzerPin = 8;   // Buzzer connected to pin 8
const int buttonPin = 2;   // Optional: Button to trigger buzzer

void setup() {
  pinMode(buzzerPin, OUTPUT);
  pinMode(buttonPin, INPUT_PULLUP); // Button to GND
  Serial.begin(9600);
  Serial.println("Arduino Buzzer Program Ready");
}

void loop() {
  if (digitalRead(buttonPin) == LOW) {
    digitalWrite(buzzerPin, HIGH); // Buzzer ON
    Serial.println("BUZZER: ON");
    delay(500);                    // Wait 500ms
    digitalWrite(buzzerPin, LOW);  // Buzzer OFF
    Serial.println("BUZZER: OFF");
    delay(250);                    // Small delay to debounce button
  }
}

How I Ran the Program

  1. Opened the Arduino IDE and copied my code into a new sketch.
  2. Connected the Arduino board to my PC using a USB cable.
  3. Set the correct board type and COM port in the IDE.
  4. Uploaded the sketch to the Arduino.
  5. Opened the Serial Monitor at 9600 baud to see messages like "BUZZER: ON" and "BUZZER: OFF".
  6. Pressed the button to trigger the buzzer and verified it made sound.

Results & Observations

After running the program:

  • The buzzer correctly sounded when the button was pressed.
  • Serial Monitor messages synchronized perfectly with buzzer ON/OFF states.
  • This helped me confirm that the code logic and wiring were correct.

This small project demonstrated the connection between code, hardware, and monitoring outputs effectively.
